Key Relationships:
Users to Pets (1 to N): One user can have multiple pets. (NB: N stands for many)
Pets to HealthRecords (1 to N): One pet can have multiple health records.
Pets to Appointments (1 to N): One pet can have multiple appointments.
Pets to LostPets (1 to N): One pet can be associated with multiple lost/found records.
Users to CommunityPosts (1 to N): One user can create multiple community posts.
Pets to ServiceProviders (N to N): Many pets can visit many service providers, and each service provider can serve many pets.
Pets to PetCareTips (N to N): Many pets can have many care tips, and each care tip can apply to many pets.
